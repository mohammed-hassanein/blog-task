فكرة المشروع

منصة SaaS لإدارة شركات السياحة والسفر من خلال واجهة موحدة، مع عزل كامل لبيانات كل شركة عبر النطاق الفرعي وإتاحة بوابة حجز عامة لكل شركة.

نموذج العمل

اشتراكات دورية شهرية أو سنوية، وحدات اختيارية Add-on، وخطة Enterprise تشمل التخصيص والدعم المتقدم.

الجمهور المستهدف

وكالات السفر، منظمو الرحلات، شركات السياحة، ومزودو خدمات السفر، مع تركيز أولي على السوق السعودي ثم الخليج والعالم العربي.

فكرة المشروع والأهداف

تهدف المنصة إلى رقمنة العمليات اليومية لشركات السياحة والسفر، وتقليل الاعتماد على ملفات Excel والجداول اليدوية، وتوحيد بيانات العملاء والحجوزات والبرامج السياحية والعمليات المالية ضمن نظام واحد.

الأهداف الاستراتيجية

  • إدارة البرامج السياحية والحجوزات والعملاء من منصة مركزية واحدة.
  • ربط الفواتير والمدفوعات والمصروفات بملفات العملاء والحجوزات.
  • توفير تقارير تشغيلية ومالية دقيقة في الوقت الفعلي.
  • بناء نظام مرن قابل للتوسع تقنياً، قالبياً، ولغوياً.
  • دعم العربية والإنجليزية واتجاهي RTL/LTR على مستوى النظام.

القرارات المعمارية الأساسية

Multi-Company

قاعدة بيانات مشتركة مع عزل منطقي صارم عبر company_id وGlobal Query Scope، وتحديد الشركة من النطاق الفرعي.

Module Architecture

كل وحدة مستقلة قابلة للتفعيل أو التعطيل لكل شركة عبر سجل وحدات مركزي.

Theming

فصل منطق العمل عن طبقة العرض، واستخدام themes/default/portal وactive_theme لكل شركة.

i18n & RTL/LTR

ملفات لغة عربية وإنجليزية، بادئة لغة للبوابة العامة، ودعم Bootstrap RTL وCSS Logical Properties.

تقويم مزدوج

تخزين ميلادي فقط، مع عرض هجري/ميلادي في جميع تواريخ النظام عند الحاجة.

حزم معتمدة

Spatie Permission مع Teams، Spatie Activity Log، Spatie Media Library، وLaravel Breeze بعد إعادة بناء واجهاته بـBootstrap.

البوابة العامة للحجوزات

لكل شركة بوابة عامة عبر نطاق فرعي خاص تتيح للعملاء تصفح البرامج السياحية وتقديم طلبات الحجز عبر الإنترنت. البوابة جزء أساسي من النواة وليست ميزة مستقبلية.

  • تعرض المحتوى حسب اللغة المختارة عبر المسارات /ar/ و/en/.
  • تستخدم القالب النشط للشركة وتحقن الهوية البصرية كمتغيرات تصميم.
  • تدعم تقديم طلبات الحجز والدفع الإلكتروني ضمن القالب الافتراضي في مرحلة البوابة.

البرامج السياحية

البرنامج السياحي مصمم كبرنامج خاص يطلبه العميل لعائلته أو مجموعته المستقلة، وليس حجزاً جماعياً مفتوحاً. الحقول القابلة للترجمة تحفظ بصيغة JSON للغتين.

المرشدون، اللغات، الشهادات، التكلفة اليومية، ومعلومات التواصل.

Activity

Restaurant

المطاعم، نوع المطبخ، الوجبات، والمدينة.

سير عمل الحجوزات

  1. استقبال الطلب: عبر البوابة العامة أو مباشرة من موظف المبيعات.
  2. تحويل البرنامج: إنشاء حجز مبدئي مع تحديد التواريخ وأعداد البالغين والأطفال.
  3. عرض السعر: إصدار عرض أو فاتورة مبدئية بناءً على التواريخ والأعداد الفعلية.
  4. تأكيد الحجز: عند استلام العربون أو السداد الكامل وتسجيل المعاملة المالية.
  5. التنفيذ: متابعة خط السير والتنسيق مع الفنادق والنقل والمرشدين والأنشطة.
  6. الإغلاق: إغلاق الحجز بعد الرحلة وتوثيق ملاحظات العميل وتقييمه.

دورة حياة حالة الحجز

Inquiry → Draft Booking → Confirmed / Deposit Paid → Fully Paid → In Progress → Completed أو Cancelled

طبقة إدارة منصة SaaS

  • إدارة الشركات وربطها بالنطاقات الفرعية وتعليق أو تفعيل الحسابات.
  • إدارة خطط الاشتراك Core وAdd-on وEnterprise.
  • سجل مركزي لتفعيل الوحدات والقوالب وتحديد active_theme.
  • فوترة اشتراكات الشركات على مستوى المنصة.
  • مراقبة صحة النظام، Horizon، ومعدلات الاستخدام.
  • الإشراف على سجلات النشاط مع الحفاظ على عزل الشركات.

خارطة الطريق وخطة التنفيذ

1

Foundation

Laravel 13، العزل بالنطاقات الفرعية، Spatie packages، i18n، RTL/LTR، القوالب، التقويم المزدوج، والمصادقة.

2

Core Modules

CRM، البرامج وخط السير، مكتبة الموارد، الحجوزات، والفواتير والمدفوعات بلغة العميل.

3

Public Portal

واجهات القالب الافتراضي، تبديل اللغة، تصفح البرامج، طلبات الحجز، والدفع عبر الإنترنت.

4

Reports & SaaS Layer

التقارير التشغيلية والمالية والإدارية، وطبقة إدارة المنصة والوحدات والقوالب.

5

Launch & Optimization

اختبارات الأمن والأداء وRTL/LTR، توثيق التشغيل، تدريب الفريق، وإطلاق تجريبي محدود.
